مقدمة عن النظام

نظام WahaManager هو نظام الإدارة والربط الوسيط بين NexusV2 وبين واجهة برمجة تطبيقات واتساب (WhatsApp API) المعروفة باسم WAHA.

يُعنى هذا النظام باستقبال الرسائل (عن طريق Webhooks)، مزامنة جهات الاتصال (Contacts Sync)، وسحب الرسائل التاريخية (Messages Sync)، بالإضافة إلى أتمتة إرسال هذه البيانات إلى نماذج الذكاء الاصطناعي (AI Analysis) لاستخراج سمات الشخصية، الاهتمامات، والموضوعات.

هيكلية الـ Frontend (واجهة المستخدم)

تم بناء الواجهة باستخدام React / Next.js في مسار app/contacts/waha-manage/page.tsx. تعمل هذه الواجهة كلوحة تحكم (Orchestration Dashboard) للتحكم الكامل بجميع العمليات.

  • Synced Contacts: يعرض العدد الإجمالي لجهات الاتصال التي تم استيرادها.
  • Ingested Messages: عدد الرسائل التي تم سحبها من واتساب إلى النظام.
  • Active AI Jobs: عدد العمليات التي تعمل حالياً في الخلفية لتحليل الرسائل باستخدام الذكاء الاصطناعي.

الـ Logic: يتم جلب هذه البيانات عبر استدعاء Polling كل 5 ثواني لـ API Endpoint: /settings/waha-manage/status لضمان تحديث الأرقام فورياً بدون إعادة تحميل الصفحة.

واجهة تعرض العمليات قيد التشغيل (مثل مزامنة الرسائل أو جهات الاتصال). تحتوي على:

  • Progress Bar: شريط تقدم يوضح نسبة إنجاز كل عملية (processed_items / total_items).
  • Pause / Resume: إمكانية إيقاف العملية مؤقتاً أو استئنافها عن طريق مناداة API خاص بها.
  • أزرار البدء: يمكن للمستخدم تشغيل Sync Contacts أو Sync Messages أو Batch Analyze AI لجميع الجهات.

نافذة تنبثق لطلب تحليل شامل للمحادثات، تتيح للمستخدم إدخال التالي:

  • Agent (Strategy ID): تحديد الوكيل أو الاستراتيجية المراد تطبيقها.
  • Model Override: تغيير الموديل الافتراضي (مثل gpt-4o).
  • Message Limit: تحديد عدد الرسائل المرجعية لتقليل استهلاك الـ Tokens.
  • Extraction Targets: تحديد ما نريد استخراجه (سمات شخصية Personality، تفضيلات Preferences، مواضيع Topics).
Business Logic: يرسل هذا الـ Form البيانات إلى /settings/waha-manage/analyze/start، والذي بدوره يُدخل العمليات في طابور (Queue) الـ Backend لكي لا يتم حظر الـ API لتخطي الحد المسموح به (Rate Limit).

هيكلية الـ Backend (الخوادم والمنطق)

معالجة الأحداث (ProcessWahaWebhookJob)

عندما يتم استقبال رسالة جديدة عبر واتساب، تقوم WAHA بإرسال Webhook إلى النظام. يتم استلامه وتحويله إلى Job يعمل في الخلفية لمعالجة الرسالة كالتالي:

  1. Resolve Contact: البحث عن جهة الاتصال بـ Phone / ChatId وإنشاءها إن لم تكن موجودة.
  2. Resolve Conversation: استرجاع أو إنشاء محادثة (Conversation) مخصصة لواتساب.
  3. Resolve Session: فتح جلسة جديدة للمحادثة أو استكمال جلسة مفتوحة.
  4. Insert Message: إدخال الرسالة إلى قاعدة البيانات وتحديث الـ unread_count وتاريخ آخر رسالة.
  5. Dispatch AI Analyze: إرسال الرسالة إلى AnalyzePeopleConnectMessageJob لكي يرد عليها الذكاء الاصطناعي أو يُحللها.
  6. Realtime Broadcasting: بث الرسالة فورياً للواجهة (عبر Laravel Reverb) ليراها المستخدم بدون تحديث.

عمليات المزامنة المتزامنة (WahaSyncProcess Model)

لأجل مزامنة البيانات القديمة (تاريخ الرسائل وجهات الاتصال) يتم استخدام موديل WahaSyncProcess. وظيفته تتبع حالة المزامنة لمنع انهيار النظام:

  • يحتفظ بحقول مثل: total_items, processed_items, status لتغذية الواجهة الأمامية بالـ Progress.
  • يحتفظ بـ last_cursor_id لكي يتم استكمال المزامنة في حال توقفت العملية (Pagination / Offset).

دورات العمل (Workflows)

Message Ingestion Workflow

1

الاستقبال (Ingestion)

يقوم مزود WAHA بإرسال Payload يحتوي على الرسالة ورقم المرسل. يتم حفظه خاماً للحماية من الضياع في PeopleConnectRawProviderEvent.

2

المعالجة في الخلفية (Background Queue)

يلتقط طابور Laravel Queue (عبر Redis) هذا الحدث ويبدأ بحقن البيانات في جداول (Contacts, Conversations, Messages).

3

الذكاء الاصطناعي والاستجابة (AI & Response)

بناءً على الرسالة، يتم استدعاء ميزة الـ LLM لتحليل نية المرسل، أو تحسين ملف تعريف العميل (استخراج صفاته)، أو توليد رد تلقائي وبثه لواجهة الويب.

تأثيرات قاعدة البيانات (Database Migrations)

لربط هذا النظام المتكامل، تم تعديل قاعدة البيانات لإضافة الحقول التالية:

Contacts Table

  • waha_contact_id: مُعرف فريد للرقم من واتساب.
  • is_waha_synced: حالة المزامنة.

Messages Table

  • waha_message_id: مُعرف الرسالة في واتساب لمنع التكرار (Duplicate Check).
  • provider_payload_hash: لحماية الـ Data Integrity.
