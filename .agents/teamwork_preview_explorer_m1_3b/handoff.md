## Caveats

- I assumed that `apiClient.get` and `apiClient.post` are exclusively used for data fetching, meaning monitoring them with `vi.spyOn` or `vi.mock` is sufficient to prove that a data-loading function was triggered.
- `app/contacts/[id]/page.tsx` is an async Next.js Server Component (using `use(params)`). It will need to be correctly mocked or wrapped during rendering in Vitest, or tested by passing a resolved Promise for `params`.
- I have not investigated the exact mock return shapes required by `apiClient` beyond what is needed to avoid unhandled promise rejections during render. Mock configurations might need refinement during actual implementation.
