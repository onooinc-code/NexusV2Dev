                         📊 QUICK FACTS
================================================================================

TECH STACK:
  Backend:   Laravel 11, PHP 8.2+, MySQL/SQLite, Redis
  Frontend:  Next.js 15, React 19, TypeScript, Tailwind CSS
  Tools:     Docker, Node.js, Composer, Git

SERVICES & PORTS:
  Frontend (Next.js):     3000
  Backend API (Laravel):  8000
  Vite Dev Server:        5173
  Reverb (WebSocket):     6001
  Redis Cache:            6379

SETUP TIME:
  First time:   5-10 minutes
  Daily:        < 1 minute (quick-start.bat)

PERFORMANCE:
  Cache speed:   50x faster (with Redis)
  Load time:     2-3x faster
  Queue:         Non-blocking, asynchronous

================================================================================
                       🚀 QUICKEST START
================================================================================

Windows (Recommended):
  1. cd NexusV2\StartNexus
  2. Double-click: quick-start.bat
  3. Wait for services to start (~30 seconds)
  4. Go to: http://localhost:3000

Command Line:
  cd NexusV2
  npm start

                       📱 MAIN FEATURES
================================================================================

✅ AI-Powered Contacts
   Deep contact intelligence, identity resolution, enrichment

✅ Autonomous Agents
   Multiple agent types (Autonomous, Reflection, Specialized)

✅ Multi-Channel Messaging
   WhatsApp, email, notifications integration

✅ Workflow Automation
   Complex automation with triggers and scheduling

✅ Memory Management

   WebSocket via Reverb, live updates

   Provider routing, cost optimization, batching

✅ Task Management
   Queue-based async job processing

================================================================================
                       🔧 DEVELOPMENT WORKFLOW
================================================================================

Daily Development:
  1. Double-click: StartNexus/quick-start.bat
  2. Edit code (hot-reload enabled)
  3. Test at http://localhost:3000
  4. Ctrl+C to stop

Adding Features:
  1. Create models, services, controllers
  2. Add routes in app/routes/
  3. Frontend components auto-hot-reload
  4. Use Horizon for queue monitoring (/horizon)

Deploying Changes:
  1. Run: StartNexus/build-all.bat
  2. Test production build locally
  3. Push to git/deploy
