AI Model Hub

Unified AI Multi-Provider Integration, Intent-Based Routing, and Circuit-Breaker Resilience System

⚡

Multi-Provider Hub

Enables Nexus V2 to scale seamlessly without coupling to a single AI vendor. Business managers can register and sync models from industry leaders or custom internal endpoints.

  • Decouples codebase from specific API vendor formats (OpenAI, Google, Anthropic, Groq).
  • Supports custom REST API wrapping for proprietary LLM deployments.
  • Simplifies new provider onboarding via dynamic forms.
🔑

Secure API Key Rotator

Ensures zero-downtime and secure token usage by abstracting and encrypting sensitive credentials, shifting them away from direct database tables.

  • Automated encryption/decryption at database boundary using AES-256-GCM.
  • Dynamic, real-time key switching depending on current executing provider.
  • Support for multiple key-header injection formats (Bearer, Key, custom).
📊

Financial Control & Budgeting

Protects the organization from runaway API expenses by placing guardrails at both global and workspace scopes.

  • Monthly limits set individually per Workspace or globally.
  • Real-time predictive forecasting engine warns when projected spend exceeds limits.
  • Automatic request blocking when hard budget caps are violated.
⚙️

Intent Routing Engine

Abstracts model details away from developers. Instead of invoking "gpt-4" or "gemini-1.5", developers call semantic intents (e.g., reasoning, general_chat, summarization).

The routing engine maps intent keys to the highest performing model configuration, automatically evaluating cost, latency, security class, and language criteria.
  • Profile Matching: Low-cost, fast-latency, high-security, or custom profile constraints.
  • Context Isolation: Transparent system-prompt injecting per route.
  • Fallback Chains: Intent configurations specify fallback providers to try if the primary fails.
🛡️

Circuit Breaker Pattern

Prevents cascading backend failures during API provider outages. Tracks health status dynamically and fails fast if the provider is down.

  • Closed State: Requests pass through normally. Failures increment a threshold tally.
  • Open State: Tally exceeded. Request fails fast, triggering immediate fallback routing.
  • Half-Open State: Tries a trial request after decay timeout. Restores Closed state on success, or re-opens on failure.

Codebase Directory Layout

  • app/Services/AiModelsHub/
    • IntentRoutingEngine.php Resolves profile constraints & maps intents to providers/models.
    • CircuitBreaker.php Protects system from API outages; executes fallback cascades.
    • PayloadAdapterFactory.php Adapts JSON inputs/outputs between generic formats and provider-specific schemas.
    • EncryptedApiKeyStorage.php Manages AES-256 key storage & database encryption boundaries.
  • app/Http/Controllers/
    • AiRequestController.php Unified POST `/api/v1/ai/request` endpoint handler.
    • AiCostAnalyticsController.php Handles budget setting and forecasting calculations.

Adaptive Payload Normalization

The PayloadAdapterFactory transforms standard payloads into API-specific shapes dynamically:

// Generic Input format
{
  "intent": "general_chat",
  "prompt": "Hello World",
  "parameters": { "temperature": 0.7 }
}

// Adapted OpenAI Output:
{
  "model": "gpt-4",
  "messages": [{"role": "user", "content": "Hello World"}],
  "temperature": 0.7
}

// Adapted Google Gemini Output:
{
  "model": "gemini-1.5-pro",
  "contents": [{"parts": [{"text": "Hello World"}]}],
  "generationConfig": {"temperature": 0.7}
}

    L --> G

    M --> N[Track Cost & Usage]
    N --> O[Store in Semantic Cache]
    O --> P[Return Response to Client]
    G --> Q[Resolve Fallback Provider]
    Q --> H
