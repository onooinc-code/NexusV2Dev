دليل التطوير بـ VSCode & Antigravity

كيف تصلح مشكلة الـ Git، تجهز بيئة العمل، وتوجه الذكاء الاصطناعي بشكل احترافي كل مرة.

1 إصلاح وتفعيل الـ Repositories الجديدة

إحساسك في محله! أنت قمت بكتابة git init و git commit ولكنك لم تقم بربط الكود بالروابط الجديدة على GitHub ولم تستخدم أوامر الـ Submodules في المجلد الرئيسي.

لتصحيح الوضع الحالي ورفع الكود بشكل نظيف، افتح الـ Terminal في المجلد الرئيسي NexusV2 وقم بتشغيل هذه الأوامر بالترتيب:

أولاً: رفع الفرونت اند

cd Nexus-Frontend
git remote add origin https://github.com/onooinc-code/NexusV2Dev_Frontend.git
git push -u origin master
cd ..

ثانياً: رفع الباك اند

cd Nexus-backend
git remote add origin https://github.com/onooinc-code/NexusV2Dev_Backend.git
git push -u origin master
cd ..

ثالثاً: تجهيز ورفع المشروع الرئيسي (Root) مع الـ Submodules

# إزالة المجلدات من التتبع العادي (لتجنب رفعها كمجلدات فارغة)
git rm -r --cached Nexus-Frontend
git rm -r --cached Nexus-backend

# إضافتهم بطريقة الـ Submodules الصحيحة
git submodule add https://github.com/onooinc-code/NexusV2Dev_Frontend.git Nexus-Frontend
git submodule add https://github.com/onooinc-code/NexusV2Dev_Backend.git Nexus-backend

# حفظ ورفع المشروع الرئيسي
git commit -m "chore: setup submodules and master project"
git remote add origin https://github.com/onooinc-code/NexusV2Dev.git
git push -u origin master

2 تجهيز VSCode كـ Multi-root Workspace

لكي لا يتلخبط Antigravity أو GitHub Copilot أو Cursor بين ملفات الفرونت والباك اند، يجب ألا تفتح مجلد NexusV2 بالكامل.

  1. افتح VSCode وقم بإغلاق أي مشروع مفتوح File > Close Folder.
  2. اذهب إلى File > Add Folder to Workspace واختر مجلد Nexus-Frontend.
  3. كرر الخطوة السابقة وأضف مجلد Nexus-backend، ثم كررها وأضف NexusDev.
  4. اذهب إلى File > Save Workspace As واحفظ الملف باسم Nexus.code-workspace في مجلد المشروع الرئيسي.

لماذا هذا مهم جداً؟ الآن الـ VSCode و Antigravity سيتعاملان مع الفرونت اند كمشروع Node.js مستقل، ومع الباك اند كمشروع Laravel مستقل، ولن يحدث تداخل في أوامر الـ Terminal أو البحث.

3 كيف تكلم Antigravity في كل جلسة تطوير؟

السر في الحصول على كود يعمل بنسبة 100% وبدون أن يخرب الـ AI المشروع هو تحديد النطاق (Scoping).

مثال 1: البدء بمهمة جديدة في الباك اند

الرسالة لـ Antigravity:
"مرحباً. أريدك أن تفتح ملف NexusDev/02_Backend_Missing_Requirements.md وتقرأ المشكلة رقم 1 الخاصة بـ (AgentsHub MCP Pivot). افتح ملفات الباك اند المتعلقة بهذه المشكلة، قم بكتابة Test يثبت وجود المشكلة، ثم قم بإصلاحها في الـ Model وتحديث الـ Migration. لا تقم بأي شيء آخر خارج هذا النطاق."

مثال 2: بناء واجهة في الفرونت اند مكملة للباك اند

الرسالة لـ Antigravity:
"لقد أنهينا لتونا تحديثات الباك اند الخاصة بـ SchedulerHub. افتح مجلد الفرونت اند الآن Nexus-Frontend. اقرأ ملف NexusDev/01_Frontend_Missing_Requirements.md المتعلق بـ SchedulerHub. أريدك أن تنشئ API Client باستخدام Axios لربط الواجهة بالباك اند الجديد، ثم قم بتحديث ملف SchedulerTab.tsx لعرض البيانات الحقيقية وتفعيل زر الإيقاف والتشغيل."

مثال 3: مراجعة الكود والرفع (Commit & Push)

الرسالة لـ Antigravity:
"الآن وقد تأكدنا أن الكود يعمل واجتاز الـ Tests، أريدك أن:
1- تحدث ملفات الوثائق في NexusV2_Docs بما قمنا به اليوم.
2- قم بإنشاء رسالة Commit احترافية بناءً على التغييرات في الباك اند."
