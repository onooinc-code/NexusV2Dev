## Local Development Setup

### Step 1: Clone Repository
```bash
git clone https://github.com/yourorg/nexus-backend.git
cd nexus-backend
```

### Step 2: Install PHP Dependencies
```bash
composer install

# If you encounter issues, try:
composer install --no-scripts
composer update
```

### Step 3: Copy Environment File
```bash
cp .env.example .env

# Edit .env with your local configuration
nano .env
```

### Step 4: Generate Application Key
```bash
php artisan key:generate

# This generates: APP_KEY=base64:xxxxx
# Verify it's set in .env
```

### Step 5: Configure Database
```bash
# Update .env with your database credentials
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=nexus_dev
DB_USERNAME=postgres
DB_PASSWORD=your_password

# Create the database
createdb nexus_dev  # Using PostgreSQL CLI
```

### Step 6: Run Migrations
```bash
# Run all migrations
php artisan migrate

# Rollback and retry if needed
php artisan migrate:rollback
php artisan migrate

# Verify migrations
php artisan migrate:status
```

### Step 7: Seed Development Data
```bash
php artisan db:seed

# Or seed specific seeders
php artisan db:seed --class=UserSeeder
php artisan db:seed --class=ContactSeeder

# List available seeders
php artisan list seed
```

### Step 8: Generate Sanctum Tokens
```bash
php artisan tinker
>>> $user = App\Models\User::first();
>>> $token = $user->createToken('dev-token')->plainTextToken;
>>> echo $token;
# Copy this token for API testing
```

### Step 9: Start Local Development Server
```bash
# Start Laravel dev server (port 8000)
php artisan serve

# Server available at: http://localhost:8000

# In another terminal, start Reverb WebSocket server
php artisan reverb:start

# Output: WebSocket server started on ws://127.0.0.1:8080
```

### Migration Structure

```php
// Example migration
Schema::create('contacts', function (Blueprint $table) {
    $table->uuid('id')->primary();
    $table->foreignUuid('user_id')->constrained()->onDelete('cascade');
    $table->string('name');
    $table->string('email')->unique();
    $table->string('phone')->nullable();
    $table->string('company')->nullable();
    $table->enum('status', ['active', 'inactive'])->default('active');
    $table->float('engagement_score')->default(0);
    $table->json('metadata')->nullable();
    $table->timestamps();
    $table->softDeletes();
    
    // Indexes
    $table->index('user_id');
    $table->index('email');
    $table->index('engagement_score');
});
```

## Queue & Job Setup

### Queue Configuration

```env
# .env
QUEUE_CONNECTION=redis

# Queue Options
QUEUE_NAME=default
QUEUE_DRIVER=redis
```

### Starting Queue Workers

```bash
# Start single worker
php artisan queue:work

# Start with specific queue
php artisan queue:work --queue=critical,default,long-running

# Start with retry settings
php artisan queue:work --tries=3 --timeout=300

# Start in background (with Supervisor)
php artisan queue:work --daemon

# Process a single job
php artisan queue:work --max-jobs=1

# Stop worker gracefully
SIGTERM signal
```

### Supervisor Configuration (Production)

```ini
# /etc/supervisor/conf.d/laravel-worker.conf
[program:laravel-worker]
process_name=%(program_name)s_%(process_num)02d
command=php artisan queue:work --queue=critical,default,long-running --tries=3 --timeout=300
autostart=true
autorestart=true
numprocs=4
redirect_stderr=true
stdout_logfile=/var/log/laravel-worker.log
stopwaitsecs=600
```

**Manage Supervisor:**
```bash
# Reread configuration
sudo supervisorctl reread

# Update configuration
sudo supervisorctl update

# Start/stop workers
sudo supervisorctl start laravel-worker:*
sudo supervisorctl stop laravel-worker:*
sudo supervisorctl restart laravel-worker:*

# Check status
sudo supervisorctl status
```

### Queue Monitoring

```bash
# List failed jobs
php artisan queue:failed

# Retry failed jobs
php artisan queue:retry [id]
php artisan queue:retry --all

# Forget failed job
php artisan queue:forget [id]

# Clear failed jobs
php artisan queue:flush

# Monitor queue length
php artisan queue:monitor redis --max=1000
```

### Test Structure

### Writing Tests

```php
// Feature Test Example
namespace Tests\Feature;

use Tests\TestCase;
use App\Models\Contact;
use App\Models\User;

class ContactTest extends TestCase
{
    public function test_can_create_contact()
    {
        $user = User::factory()->create();
        
        $response = $this->actingAs($user)
            ->postJson('/api/v1/contacts', [
                'name' => 'John Doe',
                'email' => 'john@example.com',
            ]);
        
        $response->assertStatus(201)
                 ->assertJsonStructure(['data' => ['id', 'name', 'email']])
                 ->assertJsonPath('data.name', 'John Doe');
        
        $this->assertDatabaseHas('contacts', [
            'email' => 'john@example.com',
        ]);
    }
    
    public function test_cannot_create_duplicate_email()
    {
        $contact = Contact::factory()->create();
        $user = $contact->user;
        
        $response = $this->actingAs($user)
            ->postJson('/api/v1/contacts', [
                'name' => 'Another',
                'email' => $contact->email,
            ]);
        
        $response->assertStatus(422)
                 ->assertJsonValidationErrors('email');
    }
}
```

### Production Deployment Script

```bash
#!/bin/bash
# deploy-production.sh

set -e

DEPLOYMENT_DIR="/var/www/nexus-backend"
BACKUP_DIR="/backups"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

echo "Starting production deployment..."

# Create backup
mkdir -p $BACKUP_DIR
pg_dump nexus_prod > $BACKUP_DIR/nexus_prod_$TIMESTAMP.sql

# Navigate to deployment directory
cd $DEPLOYMENT_DIR

# Maintenance mode
php artisan down --message="Deployment in progress"

# Pull code
git fetch origin
git checkout v1.0.0

# Install dependencies
composer install --no-dev --optimize-autoloader --no-interaction

# Run migrations
php artisan migrate --force

# Clear and cache configuration
php artisan config:clear
php artisan config:cache
php artisan route:cache

# Restart services
sudo systemctl restart laravel-queue
sudo systemctl restart reverb

# Exit maintenance mode
php artisan up

echo "Production deployment completed at $TIMESTAMP!"
echo "Backup saved to: $BACKUP_DIR/nexus_prod_$TIMESTAMP.sql"
```

## Troubleshooting

### Common Issues & Solutions

#### Issue: Port 8000 Already in Use
```bash
# Find process using port
lsof -i :8000

# Kill process
kill -9 [PID]

# Use different port
php artisan serve --port=8001
```

#### Issue: Database Connection Failed
```bash
# Check PostgreSQL service
sudo systemctl status postgresql

# Test connection
psql -U postgres -d nexus_dev -c "SELECT 1"

# Check .env configuration
grep "DB_" .env

# Test from PHP
php artisan tinker
>>> DB::connection()->getPdo();
```

#### Issue: Queue Jobs Not Processing
```bash
# Check queue connection
php artisan config:get queue.default

# Start queue worker
php artisan queue:work --verbose

# Check failed jobs
php artisan queue:failed

# Check Redis
redis-cli PING
redis-cli LLEN queue:default
```

#### Issue: WebSocket Connection Failed
```bash
# Check Reverb is running
ps aux | grep reverb

# Check port
netstat -an | grep 8080

# Restart Reverb
php artisan reverb:start

# Check browser console for errors
```
